Settings

OpenAI API Key

Your API key is stored locally in your browser and is never sent to any server other than OpenAI. Get a key at platform.openai.com/api-keys.

Best Models for Open Meeting Scribe

Each step of the pipeline uses a different OpenAI model optimised for that task. Defaults are pre-selected based on the recommended configuration below.

1

Live Transcript during the meeting

Transcribes each 10-second audio segment in real time. Optimised for low latency and continuous streaming.

2

Final Transcript after meeting ends

Runs once after recording stops to clean up punctuation, remove filler words, and add paragraph breaks. Handles messy audio and multiple speakers.

3

Meeting Summary structured notes generation

Generates structured notes from the final transcript: summary, key decisions, action items, and open questions. Strong structured-output reasoning is key.

Privacy

  • Meeting audio is processed entirely in memory — never written to disk.
  • Transcripts are sent directly to OpenAI from your browser using your own API key.
  • Your API key is stored in your local browser profile (encrypted at rest by Chrome).
  • No analytics, telemetry, or backend servers are used by this extension.
  • Session data is cleared as soon as you dismiss the meeting notes.
